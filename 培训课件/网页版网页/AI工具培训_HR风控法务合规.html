工具是什么、怎么选

💬 对话型 AI
网页版 Claude、ChatGPT
你粘贴内容 → 它回答 → 你复制结果
每次手动操作，适合一次性问答
🤖 智能体型 Agent
Claude Code、OpenClaw
进入文件夹 → 自己读文件 → 执行任务 → 直接输出成品
适合重复性、批量、文档密集型工作
💡
比喻：对话型是"远程顾问"，智能体型是"住在你办公室里的助理"。

四个工具对比

Cursor 图形界面
适合：需要有界面、直接看着文件操作的人
文档对比、批量填充、合同修改
Codex CLI 命令行
适合：想让 AI 在后台跑任务、处理文件的人
批量文件处理、自动化入口
Gemini CLI Google 生态
适合：多语言处理、与 Google 生态联动的人
快速摘要、中英文翻译
Claude Code 推荐
适合：需要 AI 完成复杂多步骤流程的人
文件工厂、审核自动化、Agent 大脑
非技术人员速选
我只想用 AI 写东西、改东西→Cursor 或网页版 Claude
我想让 AI 帮我处理一堆文件、自动跑流程→Claude Code
我想搭一个 24 小时自动帮我干活的系统→OpenClaw

1
监管动态日报自动生成
工具：Claude（网页版）· 无需上传文件
⏱ 3 分钟
背景：合规/风控每天需要跟进 SFC、MAS 等监管机构动态，传统方式是手动浏览官网，耗时半天整理。
▶ 复制以下内容粘贴给 Claude
请帮我搜索香港证监会（SFC）和新加坡 MAS 最近3个月关于基金代销合规的最新监管动态，
整理成一份中文简报，格式：监管机构 / 主要变化 / 生效时间 / 合规关注重点

要求：
1. 每个机构至少列出2条动态
2. 如有咨询期截止日期请特别标注
3. 最后用一句话总结本月最需要优先处理的合规事项

已验证输出

监管机构主要变化生效时间合规关注重点
香港 SFCUT Code 修订咨询，引入 VaR 衡量衍生品风险咨询截至 2026年1月21日检视现有产品风险披露分类
香港 SFC封闭式另类资产基金代销新规，明确适合性评估要求2025年2月17日起另类基金代销前须完成客户知识测试
香港 SFCFASTrack 简捷授权通道上线，审批缩短至 15 工作日2025年5月5日起符合条件基金可申请快速授权
新加坡 MASAML/CFT 通知及指引修订版生效2025年7月1日起全面检视 KYC/AML 内部流程
📢
说给受众：这份简报传统方式要半天手动整理，AI 30 秒完成。而且可以配置成每天早上自动生成，直接发到你邮箱或 Telegram，你不需要自己盯着官网。
2
文件一致性核查
工具：Claude · 需上传文件：【Demo2演示文件】基金签约包文件清单.docx
⏱ 4 分钟
背景：法务审核签约包时，基金名称、公司名称、版本号在不同文件里不一致——高频发生，容易漏，全靠人眼扫。
📎
第一步：先将 【Demo2演示文件】基金签约包文件清单.docx 上传到 Claude 对话中，再粘贴以下提示词。
▶ 复制以下内容粘贴给 Claude
请根据文件中"项目主数据"表的信息（基金名称、管理人名称、版本号），
逐一核查"文件清单"表中每行文件的：
1. 基金全称是否与主数据一致
2. 公司/管理人名称是否与主数据一致
3. 版本号是否与主数据一致
4. 是否有日期超期或其他明显异常

输出格式：文件序号 / 核查项 / 主数据 / 文件内容 / 状态（✅/❌）/ 建议处理
最后统计：共发现 X 处问题。

文件中预置的 5 处错误（AI 应全部找出）

文件核查项问题描述状态
文件 1基金全称写为"远景成长混合型基金"，缺"证券投资"❌
文件 1版本号显示 v2.1，主数据为 v2.3❌
文件 3公司名称写为"远景财富管理（香港）"，应为"远景资产管理"❌
文件 4基金全称写为"远景成长混合型证券基金"，缺"投资"❌
文件 6有效期授权委托书有效期 2023年12月31日，已过期❌
📢
说给受众：法务通常要逐份打开文件对照，几十份文件全靠人眼极易漏。AI 秒级完成，而且能处理几百份。关键：AI 找出问题，人来决定怎么处理。
3
风险揭示书初稿自动生成
工具：Claude · 素材已包含在提示词中，无需上传文件
⏱ 4 分钟
背景：风险揭示书需从 PPM、DA 等底层文件提炼关键信息。传统做法法务手动摘录，约 30 分钟/份，质量高度依赖个人经验。
▶ 复制以下完整内容粘贴给 Claude（含样本条款）
请基于以下底层文件信息，生成一份中文版风险揭示书初稿，格式要求：
1. 产品基本要素表格（基金类型、货币、认购起点、费用、赎回限制）
2. 主要风险列表，每条标注"必披露 / 策略特有 / 背景风险"三类
3. 最后单独列出你不确定的地方，供法务重点核查【AI待确认】

【PPM 关键条款摘录 — 远景成长混合型证券投资基金】
§1.2 基金类型：混合型开放式证券投资基金，权益类60%-90%，固定收益类10%-40%
§2.1 基础货币：美元（USD），可以港元认购，按前一日收盘价折算
§4.1 认购限额：最低初始认购100,000美元；最低追加认购10,000美元
§6.1 管理费1.5%/年（按月），托管费0.2%/年，绩效费超额收益20%（基准8%，高水位线）
§9.1 赎回：每季度开放一次，须提前30个自然日申请
§9.2 锁定期：首次认购后12个月强制锁定
§11.3 集中投资：可将不超过25%资产投资于单一发行人

【DA 关键条款摘录】
§4.1 赎回限制补充：特殊市场情况下可暂停赎回，通知期不少于5个工作日
     （注：DA §4.1 与 PPM §9.1 关于通知期表述略有差异，DA未明确是否适用季度赎回）
§5.2 估值方式：非上市资产按第三方独立估值，滞后期最长T+5

4
手机远程指挥，电脑后台干活
工具：Claude Code（电脑端）+ Claude 手机端 · 无需上传文件
⏱ 3 分钟
背景：你正在开会，想让 Claude 在电脑上跑一个任务。以往要等散会，现在用手机下指令，Claude 在后台执行，散会回来看结果。
第 1 步 · 电脑端（提前 30 秒）
在电脑的 Claude Code 会话中输入：
/remote-control
Claude 生成专属连接码或二维码，手机扫码即接入该会话。
第 2 步 · 手机端（现场演示）
扫码接入会话后，发送以下指令：
帮我把"待处理"文件夹里所有基金文件按文件类型分类整理，
生成清单，注明每份文件的名称、类型和是否含风险披露内容，
完成后把清单保存为"文件分类报告.docx"
第 3 步 · 把手机放进口袋，继续开会
电脑端 Claude 开始在后台处理。约 2 分钟后手机收到完成通知，电脑已生成报告文件。

合规/法务典型远程指令场景

你在哪里你说什么
客户会议室"帮我核对今天三份 KYC 材料的姓名和证件号是否一致"
机场候机"搜索今天 SFC 官网有没有新通知，有的话摘要发给我"
晚上在家"把本周所有项目签约状态汇总成一张表，明天早上我要用"
📢
说给受众：你的 AI 助理不需要你坐在电脑前。以前是"有空了才能做"，现在是"想到了说一声，回来看结果"。工作时间不再被"我得坐到电脑前"这件事限制。
🎯
四个 Demo 共同结论：AI 做的是搜索、提取、比对、生成格式化输出；人做的是判断 AI 标注的问题，决定怎么处理。省掉的是翻文件、手动对照、从零写草稿，以及"等我回到电脑前"的时间。

👥 案例 A：HR — AI 自动生成岗位 JD

问题：传统 JD 更新平均 1.5-2 小时/份，多部门协作时版本混乱，历史 JD 残留旧内容。
解决方案：三层文件夹 + 定时自动化
01_原数据/ ← 组织架构图 + 当年OKR + 历史JD参考 ↓ Claude Code 自动读取并生成 02_待审核/ ← 草稿.md（附 REVIEW_PENDING 标记） ↓ HR 只改一个字段：status: APPROVED 03_已审核/ ← 正式版 .docx + .html 自动输出

HR 全程只做一件事：判断草稿内容是否准确，改一个状态字段，系统自动归档。

场景传统耗时AI 之后
批量更新 JD（架构调整）数周1-2 天
面试题库生成2-3 小时/岗15 分钟/岗
新员工入职材料打包逐一整理自动生成个性化版本

⚖️ 案例 B：合规/法务 — 基金文件审核 AI 赋能

真实背景
法务国际基金组 3 人
每周 7-10 个项目
单个香港通道项目法务净工时 3.7-5.8 小时/人
团队反馈
"AI 更像老师而不是同事，给意见但不干活，翻译、摘抄、校对等苦活还是法务在做。"
❌ 现状（低效用法）
上传文件 → AI给建议 → 法务看完 → 法务自己去改 → 法务排版定稿
AI 做了 30%，法务还做 70%
✅ 目标（正确方向）
底层文件包+模板 → AI生成底稿+来源映射+遗漏清单 → 法务审特殊安排+关键偏离+签发
法务角色不是弱化，而是更清晰
以 2 号文件（Nominee 合同）为例
AI 读取 PPM/KTR → 强制提取关键字段 → 自动填入合同模板 → 自动校验中英文一致性 → 输出：① 合同草稿 ② 偏离点清单 ③ AI待确认项 → 法务只处理：特殊安排判断 + 偏离审核 + 最终签发
AI 负责（规则型）法务负责（判断型）
文件名/主体/版本号核查特殊安排是否接受
PPM/DA 关键条款提取与填充风险披露是否充分
中英文双语同步DA 责任条款如何取舍
版本差异比对跨区分销合规边界

🤖 案例 C：OpenClaw — 24 小时 AI 助手

OpenClaw（前身：Clawdbot / Moltbot）是近期开源社区热度最高的"本地优先" AI 智能体框架。以守护进程形式在你的设备上持续运行，可接入 Telegram/WhatsApp/Slack，读取本地文件，执行终端命令。
简单理解：雇了一个不睡觉的助理，住在你电脑里，24 小时待命，你用手机下指令。
手机 Telegram ←→ OpenClaw（家里电脑，24小时在线） ├── Bot 1：早间新闻助手 每天 7:30 自动执行 ├── Bot 2：公众号自动发布 每天 12:00 自动执行 └── Bot 3：股票盯盘助手 实时监控，触发即通知
🌅 Bot 1 · 早间新闻助手（7:30 自动推送）
抓取财经/监管新闻 → AI 提炼 5 条中文简报 → Telegram 推送。
对风控/合规特别有用：设定关键词（如"监管""处罚""新规"），有相关内容立即推送。
📝 Bot 2 · 公众号自动发布（12:00 自动执行）
12:00 AI 生成文章 → 推送到草稿箱 → Telegram 通知"草稿就绪"。人工只做一件事：点确认。
📊 Bot 3 · 股票盯盘助手（实时监控）
设定条件（如涨跌超 X%）立即推送，收盘后自动生成持仓简报，可用自然语言问"今天我的持仓怎么样"。

✅ 什么工作适合交给 AI？

高频重复的— 每周做好几次的事
有规则可循的— 有标准格式、有对照依据
文档密集的— 大量阅读、摘录、比对、翻译

❌ 什么工作不能完全交给 AI？

需要承担最终责任的决定
涉及复杂利益关系的判断
需要信任和关系的沟通
💡
一个马上可以行动的方法：今天下班前，在纸上写下你工作中最让你烦的一件重复性的事。明天试着把它描述给 AI，看看能不能帮你做掉 60% 的工作量。

附录（必读）：工具使用经验 & 敏感数据保护
用错工具，不是效率问题，是合规风险问题。在金融/法务场景使用 AI，必须先搞清楚数据去了哪里。这部分内容建议在首次使用任何工具前完整阅读。

📡 你的内容发到哪里了？

每次你粘贴文字或上传文件给 AI，内容都会被发送到对应公司的服务器处理。

Claude Code（API 调用）Anthropic API默认不用于训练 ✅法务合规首选
网页版 ChatGPTOpenAI 服务器需手动关闭设置 → 数据控制 → 关闭
Codex CLI（API 调用）OpenAI API默认不用于训练 ✅确认用企业账户
Cursor（AI 功能）Cursor + AI 提供商视模式而定开启 Privacy Mode
Gemini CLI（API 调用）Google Cloud企业 API 不用于训练 ✅使用企业 Google 账户
OpenClaw（本地运行）取决于底层 API与所调用的 API 一致
💡
关键结论：通过 API 调用（Claude Code、Codex CLI、Gemini CLI）的数据，默认不会被用于模型训练。法务/合规工作建议优先走 API 侧，而非网页对话框。

📂 敏感数据分级使用原则

数据级别例子推荐做法
公开信息监管官网内容、公开基金说明书直接给任何 AI 工具处理
内部一般信息内部流程文档、非客户相关模板API 调用工具 + 配置 .claudeignore
客户敏感信息真实姓名、证件号、账户余额必须先脱敏再提交，或使用本地模型
监管级机密处罚调查文件、内部审计报告本地模型（Ollama）或不用 AI

🎭 脱敏操作：两分钟学会

在把文件提交给 AI 前，用占位符替换敏感字段：

❌ 原文（不能直接提交）
客户张三，身份证 440301199001011234，账户余额 1,200,000 港元
✅ 脱敏后（可以提交）
客户 [客户A]，身份证 [已核验]，账户余额 [X金额] 港元
让 Claude Code 帮你自动批量脱敏：
请扫描当前目录中的所有 .docx 和 .pdf 文件，
识别并替换以下敏感字段（不删除，用占位符替换）：
- 真实人名 → [姓名已脱敏]
- 身份证/护照号 → [证件号已脱敏]
- 银行账号 → [账号已脱敏]
- 手机号 → [联系方式已脱敏]
生成一份脱敏报告，说明每份文件替换了哪些字段及数量。

Ollama 做了什么？
开源大模型（Llama、Qwen、Mistral 等）本来是几 GB 到几十 GB 的权重文件，普通人没法直接用。Ollama 把它们打包成一行命令就能下载运行的格式，并在本地起一个 API 服务监听 localhost:11434，让 Claude Code 等工具像调用云端 API 一样调用它——只是请求发到本机而不是互联网。

第一章：网页聊天 vs 本地工具，有什么区别？

在开始安装之前，先弄清楚一个很多新手会有的疑问：我已经可以在网页上和 ChatGPT / Claude / Gemini 对话了，为什么还要安装这些工具？

核心区别对比

对比维度 🌐 网页版（chatgpt.com 等） 🖥️ 本地工具（Codex / Claude / Gemini CLI）
能否读取你的代码文件 ❌ 不能，只能粘贴文字 ✅ 可以直接读取电脑上的整个项目
能否修改并保存文件 ❌ 给出答案后你要手动复制粘贴 ✅ 可以直接修改、创建、删除文件
能否运行代码 ❌ 不能 ✅ 可以在你的电脑上直接执行命令
上下文长度 有限制，长对话容易忘记前面内容 ✅ 可以加载整个代码库作为上下文
操作多个文件 ❌ 每次只能处理粘贴进去的内容 ✅ 跨文件修改，完整项目级别操作
自动化重复任务 ❌ 每次都要手动操作 ✅ 可以批量处理，脚本自动化
安装难度 ✅ 无需安装，打开浏览器即用 需要一次性安装配置
适合场景 问问题、写短文、头脑风暴 写代码、改项目、自动化工作流

一个具体的例子

假设你有一个有 50 个文件的网站项目，想把所有英文按钮文字改成中文：

🌐 用网页版的做法： 你需要手动打开每个文件，把内容粘贴到网页，让 AI 告诉你改哪里，然后再自己回去改。50 个文件就要重复 50 次，每次还要担心有没有漏掉。

🖥️ 用本地工具的做法： 你在终端里告诉 Codex："把这个项目里所有英文按钮文字改成对应的中文"。Codex 会自动扫描所有文件，批量修改，并给你一份修改清单，你只需要确认就好。

什么时候用网页版，什么时候用本地工具

  • 问一个问题、查资料、写一段文字 → 网页版就够了
  • 处理你电脑上的代码、文件、项目 → 需要本地工具
  • 想让 AI 帮你执行命令、安装依赖、运行测试 → 需要本地工具
  • 重复性工作、自动化流程 → 本地工具效率远高于网页版

💡 两者不是替代关系，而是互补。网页版用来"聊"，本地工具用来"干活"。很多人同时使用两者：用网页版讨论方案，用本地工具执行代码任务。

各平台网页版与本地工具对应关系

平台 网页版地址 本地工具名称 主要优势
OpenAI chatgpt.com Codex App / Codex CLI 最广泛使用，GPT-5 模型
Anthropic claude.ai Claude CLI（Claude Code） 复杂推理，超长上下文
Google gemini.google.com Gemini CLI 免费额度最大，多模态

第三章：安装 Codex App（桌面客户端）

Codex App 是 OpenAI 推出的桌面应用，提供可视化界面，让你像使用普通软件一样与 AI 对话写代码。它同时内置了终端，后续可以直接在里面安装其他工具。

系统要求

  • Windows：Windows 10 / 11（64 位），2026 年 3 月已正式支持
  • macOS：macOS 12 及以上，需 Apple Silicon（M1/M2/M3 芯片）
  • Linux：尚未发布，可在官网登记等待通知
  • 需要 ChatGPT Plus、Pro、Business、Enterprise 或 Edu 订阅

⚠️ 目前 macOS 仅支持 Apple Silicon（M 系列芯片）。如果你的 Mac 是 2020 年之前购买的 Intel 版本，暂时无法使用 Codex App，请改用 Codex CLI。

下载安装步骤

  1. 打开浏览器，访问 OpenAI Codex 官网：https://openai.com/codex/
  2. 找到页面上的下载按钮：
    • Windows 用户点击 "Download for Windows"
    • macOS 用户点击 "Download for macOS"
  3. 下载完成后：
    • Windows：双击 .exe 安装程序，按提示点击"下一步"直到完成
    • macOS：打开 .dmg 文件，将 Codex 图标拖到"应用程序"文件夹
  4. 安装完成后，在桌面或应用程序中找到 Codex 图标，双击打开

首次登录

  1. 打开 Codex App 后，会出现登录界面
  2. 点击 "Sign in with ChatGPT"（用 ChatGPT 账号登录）
  3. 浏览器会自动打开 OpenAI 登录页面，输入账号和密码完成认证
  4. 认证成功后，浏览器会提示返回 Codex App，点击确认即可

💡 建议使用 ChatGPT 账号登录，而不是 API Key。ChatGPT 账号登录可以使用所有功能，包括云端任务和会话历史。

选择项目文件夹

  1. 如果已有代码项目，点击 "Browse" 选择对应的文件夹
  2. 如果是第一次使用，可以先在桌面新建一个名为 my-projects 的文件夹，然后选择它
  3. 点击 "Select" 确认，进入主界面

第四章：在 Codex App 内安装 Codex CLI

Codex CLI 是命令行版本的 Codex，能在终端里直接使用。它也是后续帮我们安装 Claude CLI 和 Gemini CLI 的工具。Codex App 内置了终端，我们在这里安装 CLI，无需离开 App。

准备工作：安装 Node.js

Codex CLI 依赖 Node.js 运行环境。安装前，先检查你的电脑是否已有 Node.js：

  1. 在 Codex App 中，点击左下角的终端图标（或菜单中的 "Terminal"），打开内置终端
  2. 在终端中，直接告诉 Codex：

    "请检查我的电脑是否安装了 Node.js，如果没有，帮我安装 Node.js 18 或更高版本"

  3. Codex 会自动检查并给出操作建议，按它的提示确认即可

💡 如果想自己手动检查，可以在终端里输入 node --version 然后按回车。如果显示 v18.x.x 或更高版本，说明已安装好。如果提示 command not found，则需要先安装 Node.js。

安装 Codex CLI

方式一：直接告诉 Codex 帮你安装（推荐新手）

  1. 在 Codex App 的对话框中输入：

    "请帮我安装 Codex CLI，使用 npm 全局安装"

  2. Codex 会在终端执行以下命令，你只需点击"允许"确认： 
    npm install -g @openai/codex
  3. 等待安装完成（约 1-2 分钟），Codex 会告诉你安装成功

方式二：在终端手动输入命令

  1. 打开 Codex App 内置终端
  2. 输入以下命令并按回车： 
    npm install -g @openai/codex
  3. 等待完成后，验证安装： 
    codex --version
  4. 如果显示版本号（如 codex-cli 0.x.x），说明安装成功

首次运行 Codex CLI

  1. 在终端中输入 codex 并按回车
  2. 首次运行会提示登录，选择 "Sign in with ChatGPT"
  3. 浏览器会打开认证页面，完成后返回终端
  4. 认证成功后，Codex CLI 就可以使用了

⚠️ Windows 用户注意：如果遇到"codex 不是内部命令"的错误，请关闭并重新打开终端，或在对话框中告诉 Codex："我安装了 Codex CLI 但命令找不到，帮我解决 PATH 问题"。

第五章：用 Codex 帮你安装 Claude CLI

Claude CLI（也叫 Claude Code）是 Anthropic 公司推出的 AI 编程命令行工具，和 Codex CLI 功能类似，但基于 Claude 模型。现在我们让 Codex 帮我们安装它。

⚠️ 国内用户注意：如果你无法访问 Anthropic 官方服务，请直接跳到第六章，使用 GLM 替代方案。

让 Codex 帮你安装（推荐方式）

在 Codex App 的对话框里输入：

"请帮我安装 Claude Code（Claude CLI），使用官方推荐的原生安装方式。安装完成后告诉我如何用我的账号登录。"

Codex 会自动判断你的操作系统，并执行对应的安装命令，你只需要按提示确认每一步。

安装命令参考

Codex 会根据你的系统执行以下命令（供参考，不需要手动输入）：

macOS / Linux 系统：

curl -fsSL https://claude.ai/install.sh | bash

Windows 系统（PowerShell）：

irm https://claude.ai/install.ps1 | iex

💡 官方推荐使用"原生安装方式"（即上面的 curl/irm 命令），无需提前安装 Node.js，而且支持自动更新。如果遇到网络问题无法访问 claude.ai，可以告诉 Codex："请用 npm 方式安装 Claude Code：npm install -g @anthropic-ai/claude-code"。

登录 Claude 账号

  1. 安装完成后，在终端进入你的项目文件夹，例如：cd my-projects
  2. 输入 claude 并按回车，启动 Claude Code
  3. 首次运行会打开浏览器进行身份认证，用你的 Anthropic 账号或 Google/GitHub 账号登录
  4. 认证成功后返回终端，Claude Code 即可使用

如果还没有 Anthropic 账号，可以访问 https://claude.ai 免费注册。

第六章：国内用户方案：Claude CLI 接入 GLM 模型

在中国大陆，直接访问 Anthropic 官方服务可能遇到网络限制。好消息是：你仍然可以使用 Claude CLI 这个工具，只需把它背后调用的 AI 模型换成国内可正常访问的智谱 GLM 模型。这就是"外壳不变，换个大脑"的思路。

核心原理：Claude CLI 不等于 Claude 模型

对比维度 原版：Claude CLI + Claude 模型 国内方案：Claude CLI + GLM 模型
使用的工具界面 Claude CLI（claude 命令） Claude CLI（完全相同）
背后的 AI 模型 Anthropic Claude 系列 智谱 GLM-5 / GLM-4.7
服务器位置 美国 Anthropic 国内智谱 AI 服务器
网络要求 需访问国外服务器 无需翻墙，国内直连
账号要求 Anthropic 账号 智谱开放平台账号（免费注册）
操作方式 和原版完全一样 和原版完全一样
模型能力 Claude Sonnet/Opus（顶级） GLM-5（接近 Opus，国产顶级）
费用结构 按 API 用量付费 包月套餐，费用可控

💡 一个比喻：Claude CLI 是"驾驶舱+方向盘"，GLM 模型是"发动机"。你的驾驶方式完全不变，感受到的差异主要来自发动机性能（模型能力）。

安装配置步骤

第一步：安装 Claude CLI

如果已按第五章安装，直接跳到第二步。否则在 Codex App 里说：

"请帮我安装 Claude Code（npm install -g @anthropic-ai/claude-code），安装完成后不要启动，我需要先配置国内模型。"

第二步：注册智谱账号并获取 API Key

  1. 访问智谱开放平台：https://bigmodel.cn，点击右上角"注册/登录"
  2. 使用手机号或邮箱完成注册（无需信用卡）
  3. 登录后进入 "个人中心" → "API Keys"，点击"创建 API Key"
  4. 复制生成的 API Key 并保存（只显示一次）
  5. 根据需求购买合适的 GLM Coding Plan 套餐

第三步：一键自动配置（推荐）

在 Codex App 对话框里说：

"请在终端运行 npx @z_ai/coding-helper，帮我按照提示配置 GLM Coding Plan。"

该工具会引导你输入 API Key，自动完成所有配置，无需手动修改任何文件。

第四步：手动配置（备选）

如果自动工具不可用，让 Codex 说："帮我编辑 ~/.claude/settings.json，配置以下 GLM 环境变量，API Key 是：[你的Key]"。配置内容如下：

{
  "env": {
    "ANTHROPIC_AUTH_TOKEN": "你的GLM_API_KEY",
    "ANTHROPIC_BASE_URL": "https://open.bigmodel.cn/api/anthropic",
    "ANTHROPIC_DEFAULT_SONNET_MODEL": "glm-5",
    "ANTHROPIC_DEFAULT_OPUS_MODEL": "glm-5",
    "ANTHROPIC_DEFAULT_HAIKU_MODEL": "glm-4.5-air"
  }
}

⚠️ API Key 属于个人隐私，不要分享给他人，也不要提交到 GitHub 等公开代码仓库。

第五步：验证配置

  1. 重新打开一个新的终端窗口（重要！旧窗口不会加载新配置）
  2. 进入任意项目文件夹，运行 claude
  3. 出现提示时选择 "Yes" 确认使用该 API Key
  4. 进入后输入 /status 并回车，查看模型名称是否包含 glm 字样

GLM 与原版 Claude 的实际差异

能力维度 Claude Sonnet/Opus（原版） GLM-5（国内替代）
日常代码生成 非常强，行业标杆 接近原版，差距极小
中文理解与输出 良好 更优秀，国产模型优势明显
复杂推理与规划 Opus 非常强 GLM-5 基本持平
大型项目重构 规划能力强 可用，偶有遗漏需复查
上下文窗口大小 200K tokens 200K tokens（同等）
最大单次输出 约 32K tokens 128K tokens（GLM-5 更长）
国内网络稳定性 经常超时，需要代理 直连流畅，无需代理
费用 按量付费，用多贵多 包月固定费用，可控

💡 结论：对于绝大多数日常编程任务，使用 GLM-5 的体验与原版 Claude 差别极小。GLM-5 在中文处理和国内网络稳定性上甚至更有优势。如果你主要做国内项目，GLM 方案是强烈推荐的选择。

随时切回原版

GLM 配置和原版 Claude 配置可以随时切换。如果将来可以访问 Anthropic 服务，在 Codex App 里说："帮我把 Claude CLI 恢复到使用原版 Claude 模型"，Codex 会帮你修改配置文件。

第七章：用 Codex 帮你安装 Gemini CLI

Gemini CLI 是 Google 推出的 AI 命令行工具，基于 Gemini 2.5 Pro 模型。它提供每天 1000 次的免费使用额度，对新手非常友好。

系统要求

  • Node.js 20 及以上版本
  • 有效的 Google 账号（Gmail 即可）
  • 建议 RAM 4GB 以上（长时间使用建议 16GB）

让 Codex 帮你安装

在 Codex App 对话框里输入：

"请帮我安装 Gemini CLI（Google 的 AI 命令行工具），使用 npm 全局安装，安装完成后告诉我如何用 Google 账号登录。"

安装命令参考

npm install -g @google/gemini-cli

安装完成后验证：

gemini --version

登录 Google 账号

  1. 在终端输入 gemini 并按回车
  2. 首次运行会显示主题选择界面，选择你喜欢的主题
  3. 然后会显示认证方式，选择 "Sign in with Google"
  4. 浏览器自动打开 Google 登录页面，用你的 Gmail 账号登录并授权
  5. 认证成功后返回终端，Gemini CLI 即可使用

💡 Gemini CLI 每天免费提供 1000 次 Gemini 2.5 Pro 请求，对于日常学习和开发完全够用，不需要信用卡或付费订阅。

第八章：常见问题解答

安装类问题

Q：安装 Codex CLI 时提示"npm 不是内部命令"

原因：Node.js 没有安装，或安装后需要重启终端。解决方法：在 Codex App 对话框里说"帮我安装 Node.js"，Codex 会自动引导你完成安装。

Q：macOS 提示"无法打开，因为来自身份不明的开发者"

解决方法：右键点击 Codex 应用图标 → 选择"打开" → 在弹出的对话框中点击"打开"。这是 macOS 对未签名应用的提示，Codex 本身是安全的。

Q：Windows 安装 Claude Code 时网络无法访问 claude.ai

解决方法：在 Codex App 对话框里说"请用 npm 方式帮我安装 Claude Code"，Codex 会改用 npm install -g @anthropic-ai/claude-code 命令，通常可以正常安装。

Q：Gemini CLI 安装后运行报错"Not supported in your location"

Gemini CLI 在部分地区有访问限制。解决方法：在 Codex App 里告诉 Codex："我在中国使用 Gemini CLI 遇到地区限制，帮我配置代理或找替代方案"。

账号类问题

Q：没有 ChatGPT Plus 订阅，可以使用 Codex 吗？

Codex App 和 Codex CLI 需要 ChatGPT Plus、Pro、Business、Enterprise 或 Edu 订阅才能使用。免费版 ChatGPT 账号无法使用 Codex。Gemini CLI 有每日免费额度，推荐作为备选。

Q：Anthropic 账号怎么注册？

访问 https://claude.ai，点击"Sign Up"，可以用 Google 账号或邮箱注册，完全免费。

使用类问题

Q：安装完成后，每次使用要打开哪个软件？

日常使用推荐打开 Codex App，它提供最完整的图形界面体验。如果更喜欢纯命令行，也可以直接在系统终端运行 codex、claude 或 gemini 命令。

Q：这几个工具有什么区别，我应该用哪个？

工具 最适合场景 特点
Codex App 新手入门、管理多个任务 图形界面，最易上手
Codex CLI 终端操作、脚本自动化 与 App 同模型，更轻量
Claude CLI 复杂代码分析、长上下文 推理能力强，支持 MCP
Gemini CLI 日常免费使用、多模态 每日 1000 次免费，超大上下文

Q：Claude CLI 接入 GLM 模型后，跟原版 Claude 有多大差距？

对于日常编程任务，差距极小。GLM-5 在编程基准测试中接近 Claude Opus 水平，中文能力和国内网络稳定性反而更好。主要差距在于：复杂多步骤的大型重构任务中，原版 Claude Opus 的规划能力略强；以及部分多模态输入支持尚不完整。

Q：GLM 方案配置完后，Claude CLI 的所有功能都能用吗？

绝大多数功能可以正常使用，包括：文件读写、代码生成、终端命令执行、MCP 协议支持等。极少数依赖 Anthropic 特有能力的功能（如某些官方 Skills）可能不可用。

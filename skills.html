官方 Skills 指南

Skills 技能包 — 用 AI 的专业模式处理你的工作

这里列出的都是 Claude Cowork 官方提供的真实 Skills，通过插件库即可找到和启用，不是自定义脚本，不需要找开发。每个 Skill 都是针对特定工作场景预配置好的专业模式：格式、流程、输出标准全部内置，说出你要做什么，AI 自动调用。

✅ 官方可查，插件库直接安装 16 个精选 Skills 全员均有适用的场景
🧩
什么是 Skills？

Skills 本质上是给 AI 装的“能力扩展包”。没有 Skill 时，AI 只是一个通用助手；装上某个 Skill 后，AI 就学会了一套固定工作流程，知道该按什么步骤处理任务、该输出什么格式、哪些地方要提醒你人工复核。你可以把它理解成：给 AI 安装一个专门处理某类工作的 App。

📱

它是什么

Claude 本体像一台空白手机，先有基础能力；每个 Skill 像一个 App。每装一个 Skill，AI 就多学会一项具体工作技能，比如合同审查、报表整理、会议纪要、数据看板。

🧱

它里面有什么

一个 Skill 背后通常是一组预先写好的规则。最核心的是 SKILL.md，它定义名称、适用场景、工作流程和输出要求；有时还会带脚本、模板或参考资料。

▶

它怎么被用起来

你不用研究代码，也不用理解 Skill 文件。平时只需要说出任务，比如“帮我审查这份合同”或“把这份会议录音整理成纪要”，AI 会自动调用合适的 Skill；需要精确指定时，再用 $skill名称。

Skill 的结构说明：一个 Skill 通常就是一个文件夹
你可以把它理解成“一个任务场景 + 一份说明书 + 一些可选辅助材料”。其中 SKILL.md 是必须的，其它目录按需存在。
文件夹式打包

目录结构示意

contract-review-assistant/
├── SKILL.md
├── scripts/
├── references/
└── assets/
必须项

SKILL.md：Skill 的核心说明文件。它告诉 AI 这个 Skill 叫什么、什么时候用、怎么做、最终要输出什么格式。

可选项

scripts/：可执行脚本。适合做重复、机械、规则明确的动作，比如格式转换、批量检查、数据清洗。

可选项

references/：参考资料。可以放制度、模板、示例、口径说明，让 AI 按统一标准处理任务。

可选项

assets/：资源文件。常见是图片、图标、样式、附件等，用来补充 Skill 的展示或生成结果。

一句话理解：Skill 不是一段临时提示词，而是一个“可复用的工作包”。AI 先读 SKILL.md 知道该怎么做，再按需调用脚本和参考资料，最后把结果整理成固定格式。
例子：一个“合同审查 Skill”实际会做什么
示例 Skill：contract-review-assistant
你怎么说

一句自然语言就够了

比如你对 AI 说：

“帮我审查这份 IT 服务合同，标出风险条款，并和公司标准模板对比。”

Skill 在后台怎么做

它会按预设流程工作

  • 提取合同金额、期限、违约责任等关键条款
  • 对条款做红 / 黄 / 绿三级风险标注
  • 和公司标准模板做偏差对比
  • 最后生成法务审核意见摘要
你会拿到什么

输出不是一句话，而是一套结果

  • 合同基本信息表
  • 关键条款清单和风险等级
  • 偏差对比表
  • 审核意见摘要
这就是 Skill 和普通聊天的区别：普通聊天可能只给你一段泛泛建议；调用 Skill 后，AI 会按固定流程产出更接近真实工作交付物的结果。但它仍然是 AI 草稿，尤其法务、合规、风控类内容，最后必须由专业人员审核。
💡
一句话记忆：Skill 不是“一个更长的提示词”，而是“给 AI 预装好的一整套工作方法”。你负责提出任务，Skill 负责把处理步骤和输出结构固定下来。
💡
为什么要用 Skills？
1

不用每次重新解释背景

没有 Skill：你每次都要说"你是一个财务分析师，我们的科目分类是……"。有 Skill：直接说"帮我做本月的差异分析"，格式和逻辑已经内置好了。

2

输出格式直接可用

每个 Skill 针对场景预设了输出格式，比如账户核对的表格、合规追踪的检查列表、流程文档的 RACI 矩阵，不需要你再重新整理格式。

3

官方维护，持续更新

Cowork 官方 Skills 由专业团队维护，随着模型升级同步优化，不用担心提示词过时失效。出了问题有官方支持，不是靠个人摸索。

4

全团队用同一套标准

IT 统一安装，全员用相同的 Skills，输出风格和格式保持一致，不会出现每个人用法不同、输出结果差异很大的情况。

▶
怎么触发 Skills？

🗣 方式一：自然语言（推荐所有人）

直接描述你要做的事，Claude 自动识别并调用对应 Skill。不需要记任何命令。

# 说出任务，AI 自动匹配 "帮我做一份月末账户核对" → 调用 finance:reconciliation "整理这个流程的 SOP 文档" → 调用 operations:process-doc "帮我做一个数据看板" → 调用 data:build-dashboard

⌨️ 方式二：直接指定（进阶用法）

知道 Skill 名称时，可以在对话开头用 $skill名称 精确调用。

# 格式：$分类:技能名 $finance:reconciliation $operations:compliance-tracking $anthropic-skills:xlsx # 查看当前所有已安装 Skills /skills
💡
调用 Skill 时，AI 通常会在输出前标注所使用的 Skill 名称，方便你确认是否匹配到了正确的技能。
📥
怎么开启 Skills？
✅
如果 IT 已经帮大家统一安装了：直接用，不需要做任何操作。下面的步骤是给需要自己安装或补充插件包的同事。
1
打开 Claude Cowork → 设置（Settings）
点击左下角的设置图标，进入 Plugins（插件） 选项卡。这里列出了当前已安装的所有插件包，每个插件包里包含若干个 Skills。
2
在插件市场搜索并安装对应插件包
点击 Browse Plugins（浏览插件），搜索下方表格中对应的插件包名称，点击 Install 安装。安装完成后，该包里的所有 Skills 立即可用。
插件包包含 Skills适合人群
anthropic-skillsdocx · pptx · xlsx · pdf · schedule全员
knowledge-work-plugins/financereconciliation · variance-analysis · audit-support · close-management财务团队
knowledge-work-plugins/operationscompliance-tracking · risk-assessment · process-doc · process-optimization · vendor-management合规/风控/运营
knowledge-work-plugins/dataanalyze · build-dashboard · sql-queries · data-exploration数据/分析岗位
3
直接开始用，无需重启
插件安装后立即生效。在对话框里说出你的任务，Claude 会自动调用对应 Skill。输入 /skills 可以查看已安装列表，确认安装成功。
⚠️
找不到插件市场入口？ 联系 IT 同事，请他们在组织管理后台统一部署插件包，这样全员账号都会自动获得对应 Skills，不需要每个人单独安装。

常见问题
Q1这些 Skills 现在就能用吗？
如果 IT 已经给公司账号统一安装了对应插件包，直接在 Cowork 里用自然语言触发即可。如果还没有，按上方"怎么开启"一节的步骤自行安装，或联系 IT 协助统一部署。
Q2一个任务可以同时用多个 Skills 吗？
可以。Claude 会根据任务内容判断是否需要组合使用 Skills。比如"帮我把这份差异分析做成 Excel 报表"，可能同时调用 finance:variance-analysis 和 anthropic-skills:xlsx，你不需要单独指定。
Q3怎么知道 AI 用了哪个 Skill？
Claude 调用 Skill 时，通常会在回复开头标注所使用的技能名称（如"正在使用 finance:reconciliation"）。也可以随时输入 /skills 查看当前已安装和启用的列表。
Q4Skill 的输出可以直接用吗？
所有 Skill 输出均为 AI 草稿，仅供参考。涉及财务、法务、合规、风控等专业领域，必须经过相关专业人员审核确认后再正式使用。AI 负责起草，人负责判断。
Q5我的业务场景没有对应的官方 Skill，怎么办？
三条路径：① 先试用最接近的 Skill，看是否够用；② 联系 IT/AI 团队评估是否定制内部 Skill；③ 直接用自然语言描述你的需求，Claude 在没有 Skill 的情况下也能处理大多数任务，只是需要你把背景和格式要求说得更清楚。
Q6Skills 会用到我的数据做训练吗？
根据 Anthropic 企业版政策，Claude for Work 默认不会将用户输入用于模型训练。具体数据使用条款以公司与 Anthropic 签署的企业协议为准，如有疑问请联系 IT 或法务确认。

Codex App 是什么

Codex App 是 OpenAI 推出的 AI 助手应用。虽然名字里有 “编程”，但它并不只会写代码。对 HR、法务、风控、合规、运营这些岗位来说，它更像一个能看文件、整理资料、改文档、跑流程、生成初稿的智能助理。

通俗理解

它像一个全能型智能助理

你用自然语言告诉它要做什么，它就能帮你处理文档、查资料、整理表格、批量重命名文件、起草邮件和报告。区别在于，它不是只会说建议，而是真的能在你的电脑或工作区里动手完成一部分任务。

适合谁用

非技术同事优先用桌面版或网页版

如果你不想碰命令行，推荐直接使用 Codex App 桌面版或网页版。界面更像普通办公软件，上手成本低；CLI 和 IDE 更适合技术同事。

桌面版 网页版 CLI IDE
使用方式适合谁理解方式
macOS 桌面应用非技术同事优先像打开微信或飞书一样直接使用
网页版不想安装本地软件的人打开浏览器就能用
CLI有一定技术基础的同事更像命令行工作台
IDE 扩展开发同事直接集成在代码编辑器里
💡
一句话理解：ChatGPT 更像一个会回答问题的顾问；Codex App 更像一个能进入工作区、按步骤帮你把任务做出来的助理。

10 个高频技巧
Tip 1

复杂任务先开 Plan Mode

Plan Mode 会先让 AI 给出计划，再等你确认后执行，适合多步骤、多人确认、方向容易跑偏的任务。

/planShift + Tab
Tip 2

简单任务直接 Fast Mode

像格式转换、简单文案修改、快速查信息、表格清洗这类一两句话能说清的任务，直接让它动手更省时间。

Tip 3

自然语言越具体越好

最实用的提示词公式是：目标 + 受众 + 风格 + 长度 + 特殊要求。越具体，返工越少。

Tip 4

把 Skills 当成工作 SOP

Skill 像标准操作流程。装好之后，AI 会按固定步骤输出更像真实交付物的结果。

查看 Skills 指南
Tip 5

重复任务交给 Automations

适合每周周报、每天邮件摘要、定期文件检查、月度财务整理这类时间驱动的工作。它做完后会进入审查队列，还是你拍板。

周报初稿邮件摘要文件巡检
Tip 6

用 Threads 管理并行任务

每个线程就是一个独立任务窗口。把“合同审查”“会议纪要”“报表处理”分开建线程，会比把所有事塞在一个对话里清楚得多。

Tip 7

语音输入适合复杂需求

当你需要快速描述多步骤任务、在移动设备上使用、或者头脑风暴时，语音输入往往比打字更顺。

Tip 8

模型级别按任务强度选

低级别更快更省，高级别更强但更慢。不要所有任务都用最高档，简单任务没必要。

Tip 9

审查 AI 输出，不要直接信

特别是法务、合规、财务、风控场景。你要看事实是否准确、口径是否合规、措辞是否合适、有没有遗漏关键信息。

Tip 10

从小任务开始，逐步升级

先从文案修改、格式转换、信息查询开始，再到文档撰写、数据整理、自动化和多 Agent 协同，不用一步到位。

❌ 模糊描述✅ 更好的说法
帮我写个通知帮我写一份面向全体员工的年假调整通知，语气正式但友好，300 字以内，包含新旧政策对比。
检查一下合同请审查这份采购合同的付款条款，重点关注违约金比例是否合理，并列出潜在风险点。
整理数据将这份 Excel 中的客户数据按省份分组，计算每个省的客户总数和平均订单金额，输出为表格。
任务类型推荐级别原因
简单文案修改、格式调整低级别速度快、节省额度
文档撰写、邮件回复中级别平衡质量与效率
复杂分析、合同审查、策略制定高级别需要更强推理能力
阶段任务示例预计时间
入门简单文案修改、格式转换、信息查询第 1 周
进阶文档撰写、会议纪要、数据整理第 2-3 周
熟练合同审查、多步骤分析、自动化设置第 4-6 周
精通多 Agent 协同、自定义 Skill、复杂工作流第 7 周起
📝
提示词公式：目标（做什么） + 受众（给谁看） + 风格（什么语气） + 长度（多少字） + 特殊要求（格式、重点等）。大部分办公场景都适用。
🪜
模式选择小技巧：如果任务简单到你能一句话说清楚，直接用 Fast Mode；如果需要两句话以上才能描述清楚，优先切到 Plan Mode。
Automations 的类比

像给助理设闹钟

你把时间和任务先设好，到点它自动去做，做完把结果放进 Review Queue 等你审。你不满意可以退回，满意了再批准。

Threads 的类比

像多个聊天窗口

每个线程只关注当前任务，互不打扰。这样做的好处是上下文不会混掉，后面也更容易找回历史过程。

⚠
重要提醒：AI 是强大的助手，但不是最终裁决者。尤其在法务、合规、财务、风控等专业场景，把它当“第一稿提供者”，不要当“最终拍板人”。
🖥️
它如何操作你的电脑
🛠
关键区别：Codex 不只是聊天。它能读取和编辑文件、运行命令、生成文档、批量处理资料。你可以把它理解为“会先解释、再动手、必要时向你审批”的数字助理。
能力类别具体操作对你意味着什么
文件操作读取、创建、编辑、移动、删除文件帮你整理文件夹、批量重命名、归档文档
文档处理创建和编辑 PDF、Word、Excel、PPT帮你做报表、写文档、排版幻灯片
运行程序执行脚本、测试运行器、包管理器帮你自动跑数据处理流程
Git 版本控制查看变更、暂存、提交、推送帮你管理文件修改历史
终端命令在内置终端中执行各种命令相当于替你操作命令行
网络搜索内置搜索和资料检索帮你查资料、找参考信息
图像处理生成、编辑图像帮你做海报素材、UI 示意图
工作模式说明类比适合谁
Local直接在当前文件夹工作，速度快，但改动会直接影响原文件。助理直接在你办公桌上处理文件熟悉工具后的小范围改动
Worktree在隔离副本中工作，不碰原始文件。先在草稿纸上写好，再给你确认新手默认推荐
Cloud在云端环境运行，不占本机资源。助理在另一间办公室远程帮你做大任务、长任务、远程执行
沙箱级别说明类比推荐场景
read-only只能看，不能改，不能跑命令。助理只能看你桌上的文件，不能动初次尝试、只查阅资料
workspace-write可以在指定工作区读写和运行常规命令。助理可以在指定区域内自由工作日常推荐默认值
danger-full-access几乎无限制，风险最高。给助理配了全部钥匙只在完全信任场景下用
审批机制

你始终是最终决策者

当 Codex 需要执行超出沙箱范围的操作时，会暂停并请求你批准。你会看到三种常见选项：

  • 批准一次：只允许这一次操作
  • 本次会话批准：本轮对话都允许类似操作
  • 拒绝：不允许执行
内置终端

不用学命令行也能用

Codex App 内置终端，可以用 Cmd+J 打开。你不需要自己敲命令，Codex 会先解释它要执行什么、为什么执行，再等你批准。

场景 1

批量整理文件

你说：“把 download 文件夹里所有 PDF 按日期分类。” 它会读取文件、分组、创建子目录并移动文件。

场景 2

Excel 数据处理

你说：“按省份汇总客户数据，计算平均订单金额。” 它会读表、分析结构、执行计算，并生成新表。

场景 3

批量格式转换

你说：“把 20 份 Word 全部转成 PDF。” 它会遍历文件、逐个转换并存到新目录。

场景 4

根据数据做 PPT

你说：“根据季度销售表生成汇报 PPT。” 它会读数据、生成图表、创建页面并按模板排版。

⚠
给非技术同事的安全建议：首次使用优先选 Worktree 模式，保持默认的 workspace-write 权限；看到审批提示先读清楚它要做什么，不确定就拒绝；重要文件先备份再让它处理。
🛡
进一步的安全使用要点：不确定时选最小权限范围；先选“批准一次”，只有确定安全且频繁需要时再放宽；长任务时可以开启“保持电脑唤醒”，避免电脑休眠中断任务。
👥
多 Agent 协同

多 Agent 协同的本质，是让多个 AI 助手并行处理不同子任务，最后把结果汇总起来。你可以把它理解为：一个项目经理，把同一项复杂工作拆给几位不同分工的同事同时推进。

场景一

并行审查同一份合同

Agent A 看条款风险，Agent B 看合规性，Agent C 看格式规范。三份结果一起回来，比一个人全看更快。

场景二

批量处理

比如一批简历、一批风控预警、一批文档格式化任务，按批次或按文件拆给多个 Agent 同时做。

场景三

内容创作流水线

Agent A 起草，Agent B 复核准确性和合规性，Agent C 负责润色排版，最后你只审最终版。

场景四

分头研究再汇总

多个 Agent 分别调研不同主题或市场，最后由一个汇总 Agent 把结果合并成完整报告。

内置角色特点适用场景
default全能型，能处理各种任务日常工作，不确定选谁时
worker偏执行，效率优先文档处理、数据整理、明确执行类任务
explorer偏探索和收集信息调研、分析、资料汇总
类比理解

像项目经理分工

你不会一个人做完所有事，而是把法务、财务、研究这些子任务分给不同成员并行推进。多 Agent 协同就是这个逻辑。

角色分工

像公司里的不同岗位

给不同 Agent 明确分工，相当于给每个岗位写清 JD。这样每个 Agent 都是某个环节的“专人负责”。

自定义 Agent

打造你的专属团队

除了内置角色，你还可以为不同工作场景创建专属 Agent，比如合同审查专家、HR 招聘助手、合规检查 Agent、风控预警 Agent。这样每个 Agent 都会带着固定背景和专业规则来工作。

批量处理

CSV 驱动的多 Agent

如果你把一批待办事项整理成 CSV，系统可以按行分配 Agent 同时处理。适合 100 份简历初筛、50 份合同初筛、200 条预警批量分析这类高重复任务。

日常管理

如何管理多个 Agent

  • 查看状态：运行中、暂停、已完成
  • 引导或停止：中途追加说明，或直接停止某个 Agent
  • 审查结果：所有输出都会进入 Review Queue 等你审阅
  • 通过 /agent 快速切换到指定线程
📌
使用多 Agent 时记住三件事：额度消耗会更高；建议从 2-3 个 Agent 起步；任务拆分质量决定效果，拆得越清楚，结果越稳定。
⌨️
常用斜杠命令速查
命令作用使用场景
/plan切换到计划模式复杂任务先规划，再让 AI 执行
/status查看当前状态确认当前模式、模型级别、Agent 配置
/personality切换 AI 风格选择简洁或详细的交互风格
/agent切换 Agent 线程多 Agent 模式下快速跳转
/experimental启用实验功能体验测试中的新能力，谨慎使用
💡
不用死记：在输入框里打一个 /，命令列表就会弹出来。对大多数同事来说，记住 /plan、/status、/agent 就够用了。
